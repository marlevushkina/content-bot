# Content Bot Setup Guide

Пошаговая инструкция по настройке content-bot для генерации контент-планов и seeds.

---

## 🚀 Быстрый старт

### 1. Запустите интерактивный setup

```bash
chmod +x scripts/setup.sh
./scripts/setup.sh
```

Скрипт задаст вопросы и автоматически настроит бота под вас:
- Имя, канал, город
- Команда и проекты (опционально)
- Основные темы контента
- Примеры постов для tone-of-voice

### 2. Доработайте контент-стратегию

После setup откройте файлы и доработайте:

**`vault/.claude/skills/content-seeds/references/strategy.md`**
- Распределите процентное соотношение тем
- Добавьте recurring themes (темы о которых пишете регулярно)
- Опишите форматы контента

**`vault/.claude/skills/content-seeds/references/tone-examples.md`**
- Добавьте больше примеров ваших постов (минимум 3-5)
- Лучше брать посты с высоким engagement
- Разные форматы: длинные, короткие, треды

**`vault/.claude/skills/content-seeds/references/humanizer.md`** (опционально)
- Скопируйте из примеров если хотите избежать AI-паттернов
- Или создайте свой список запрещённых фраз/конструкций

### 3. Запустите бота

```bash
# Настройте Telegram бота (см. основной README)
# Запустите команду /content

/content  # генерация seeds из weekly материала
/plan     # создание контент-плана на неделю
```

## 🎨 Настройка tone-of-voice

Качество генерируемого контента зависит от примеров постов.

### Какие посты использовать как примеры?

✅ **Берите:**
- Посты с высоким engagement (много просмотров/реакций)
- Разные форматы: длинные (800+ слов), короткие (150 слов), треды
- Посты в которых максимально проявляется ваш голос
- Личные истории, кейсы, инсайты

❌ **Не берите:**
- Репосты чужого контента
- Новостные анонсы без вашего комментария
- Слишком сухие или формальные посты

### Формат tone-examples.md

```markdown
# Tone of Voice Examples

## Пример 1: Длинный пост про делегирование

[Полный текст поста]

**Почему этот пример:** личная история, уязвимость, конкретный кейс

---

## Пример 2: Короткая мысль

[Текст]

**Почему этот пример:** разговорный язык, эмоджи как реакция, не как украшение

---

## Пример 3: Тред про AI

Часть 1: [текст]
Часть 2: [текст]
...

**Почему этот пример:** структура треда, переходы между частями
```

## 🎯 Настройка контент-стратегии

Откройте `vault/.claude/skills/content-seeds/references/strategy.md`

### 1. Контент-микс (процентное соотношение)

Определите сколько процентов каждой теме:

```markdown
## Контент-микс

- Psychology/Mindset: 40%
- Systems & Operations: 30%
- Case Studies: 20%
- Life/Personal: 10%
```

Это поможет боту балансировать темы при генерации seeds.

### 2. Recurring Themes (повторяющиеся темы)

Темы о которых пишете регулярно:

```markdown
## Recurring Themes

- **Делегирование** — еженедельные инсайты про найм и управление командой
- **AI-инструменты** — обзоры новых AI-сервисов, кейсы внедрения
- **Кейсы из практики** — stories from the trenches
- **Менторство** — анонимизированные кейсы менти
- **Build in public** — прогресс по проектам (revenue, metrics)
```

### 3. Нарративные арки (опционально)

Если планируете многосерийные посты:

```markdown
## Арки (5-7 постов)

### Арка 1: AI-трансформация фаундера
1. Setup — как было раньше (ручная работа)
2. Inciting incident — первая встреча с AI
3. Эскалация — внедрение во все процессы
4. Кульминация — момент "щелчка"
5. Резолюция — новый способ работы

### Арка 2: ...
```

## ⚙️ Опциональная настройка

### Humanizer (антипаттерны AI-стиля)

Если хотите избежать "ChatGPT-измов":

1. Создайте `vault/.claude/skills/content-seeds/references/humanizer.md`
2. Добавьте запрещённые фразы и конструкции:

```markdown
# Humanizer — Антипаттерны AI-стиля

## ❌ Запрещённые конструкции

### Параллелизм
- "Не X. А Y." → слишком написанно
- "Она не мотивирует. Она парализует." → AI-паттерн

### Клише
- "важно понимать", "следует отметить" → канцелярит
- "на самом деле", "это про..." → избитые переходы

### Эмоджи
- ✨ в начале абзацев → украшение (плохо)
- 😅 после фразы → реакция (хорошо)

## ✅ Что использовать вместо

- Разговорный язык: "офигеть", "че-то", "блин"
- Скобочные мысли: (кстати...), (!!)
- Короткие пробивные: "Ну и вот оно :)"
```

### Formatters (для Telegram-бота)

Если интегрируете с Telegram ботом, используйте `src/content_bot/bot/formatters.py`:
- Форматирует длинные отчёты в HTML
- Делит на части по 4000 символов
- Сохраняет структуру с заголовками

## 🧪 Тестирование

### 1. Проверьте что placeholders заменены

```bash
# Не должно быть [Your Name] и других placeholders
grep -r "\[Your" vault/.claude/skills/
```

Если нашлись — setup скрипт пропустил их. Замените вручную.

### 2. Запустите генерацию seeds

Через Telegram бота или напрямую через Claude Code:

```bash
# Положите raw материал в vault/daily/YYYY-MM-DD.md
# Запустите skill

claude -p "Skill: content-seeds. Сгенерируй seeds из материала за неделю"
```

### 3. Проверьте стиль

Прочитайте сгенерированные hooks:
- Звучат как вы?
- Нет AI-конструкций?
- Эмоджи используются естественно?

Если нет — добавьте больше примеров в `tone-examples.md`.

## 🔧 Troubleshooting

### "Бот генерирует не в моём стиле"

→ Добавьте больше примеров постов в `tone-examples.md` (минимум 5, лучше 10+)
→ Убедитесь что примеры разнообразные (длинные/короткие, разные темы)
→ Проверьте что `humanizer.md` настроен под ваш язык

### "Seeds не соответствуют моим темам"

→ Доработайте `strategy.md` — укажите чёткие recurring themes
→ Проверьте контент-микс (процентное соотношение)
→ В raw материале (daily записи) должно быть достаточно контента по вашим темам

### "Placeholders [Your Name] остались в файлах"

→ Запустите `./scripts/setup.sh` повторно
→ Или замените вручную через `sed`:

```bash
cd vault/.claude/skills/
sed -i 's/\[Your Name\]/Ваше Имя/g' */SKILL.md
```

### "Не знаю что писать в strategy.md"

→ Посмотрите на последние 20-30 постов вашего канала
→ Сгруппируйте по темам — какие темы повторяются?
→ Это и есть ваши recurring themes
→ Посчитайте процентное соотношение — это контент-микс
